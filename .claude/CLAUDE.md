# Project Instructions

## Stack
- **Frontend**: React / Next.js + Tailwind CSS
- **Backend**: Node.js / PHP
- **Database**: SQLite3
- **Languages**: JavaScript, PHP, CSS

## General Rules
- Always write clean, readable code
- Use meaningful variable and function names
- Add comments for complex logic
- Follow existing code style in the project
- Never delete files without confirmation
- Use JavaScript only, NO TypeScript

## Database
- Database: SQLite3
- Always use parameterized queries to prevent SQL injection
- Always handle errors gracefully
- Backup before running migrations

## Git
- Write clear commit messages in English
- Use conventional commits: feat, fix, docs, refactor, test
- Never commit sensitive data or API keys

## Behavior Rules
- Do NOT use extended thinking or reasoning mode
- Answer directly and immediately
- For simple tasks like reading folders, use tools directly without planning
- Do not choreograph or plan before acting
- Always use low effort mode, never high or medium

## File Reading Rules
- When reading folders, show ALL files regardless of extension
- Include ALL extensions: .js .jsx .cjs .json .txt .md .sh .env .css .html .ts .tsx and any other
- Never filter or skip any file type
- Show the complete list, do not summarize or truncate
- When creating files in myApp_v2, preserve the exact same extension as the original file

## Format Rules (ONLY when creating or editing files)
- Use JavaScript only, no TypeScript
- Use .js and .jsx extensions only instade of .ts and .tsx
- Follow existing code style in the project
